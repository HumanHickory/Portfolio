Fun Font Sizes with Ren!

The Pixel (px)

Details

Common Sizing Unit: 16px
User Customizable: No
Changes with Zoom: No
Changes with Parent: No
Details: Pixel sizing is absolute. If a user changes the root size of the font in their browser, anything with a px unit will be unchanged. This is developer friendly, as you do not need to worry about root sizes changing the look of your layout. However, this is not user friendly, as your font may be larger or smaller than they desire.

Examples

16px: Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
24px: Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

Parent Element Responsiveness

The following box will be styled to a 20px font size. The pixel element does not change when put into a parent element with an added font-size.

Control Text: I am unstyled; I have a font-size of 20px.
Unit Text: I am styled to 16px. I do not change sizes inside a parent element.

The Root Element (rem)

Details

Common Sizing Unit: 1rem
User Customizable: Yes
Changes with Zoom: No
Changes with Parent: No
Details: REM is based on the size of the root size of the browser. In most cases, the default of the root-size is 16px. In that case, 1rem = 16px. However, if the user changes their root size to 12px, then 1rem = 12px.

Examples

1rem: Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
1.5rem: Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

Parent Element Responsiveness

The following box will be styled to a 20px font size. The rem element does not change when put into a parent element with an added font-size.

Control Text: I am unstyled; I have a font-size of 20px.
Unit Text: I am styled to 1rem. I do not change sizes inside a parent element.

The Element Parent (em)

Details

Common Sizing Unit: 1em
User Customizable: Yes
Changes with Zoom: No
Changes with Parent: Yes
Details: EM is based on the size of the parent element. By default, the em will equal the root size of the browser (which is the same as rem). However, when put into a div or other parent element that has been pre-styled, 1em will equal that font-size. Therefor, a class with 1em will not necessarily render the same througout the page.

Examples

1em: Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
1.5em: Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

Parent Element Responsiveness

The following box will be styled to a 20px font size. The em unit is dependent on the parent font-size.

Control Text: I am unstyled; I have a font-size of 20px.
Unit Text: I am styled to 1em. Because the parent element is 20px, 1em now equals 20px.

The View Width (vw) and View Height (vh)

Details

Common Sizing Unit: 1vw, 1.5vh
User Customizable: No
Changes with Zoom: Yes
Changes with Parent: No
Details: Viewports are unique in that they change size with zoom.
  • 1 vw = 1/100th viewport width.
  • 1 vh = 1/100th viewport height.
Viewports will maintain their size even when users zoom in and out. This can be frusterating for users if they think the text is too big or too small.

Examples

1vw: Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
1.5vw: Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
1 vh: Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
1.5vh: Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

Parent Element Responsiveness

The following box will be styled to a 20px font size. The viewports will not change with parent sizes.

Control Text: I am unstyled; I have a font-size of 20px.
Unit Text: I am styled to 1vw.
Unit Text: I am styled to 1vh.

The Percentage (%)

Details

Common Sizing Unit: 100%
User Customizable: Yes
Changes with Zoom: No
Changes with Parent: Yes
Details: Percent as a unit is flexible like em. It will take into account the root size, and a parent div size. It is very reliable and consistent.

Examples

100%: Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
150%: Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

Parent Element Responsiveness

The following box will be styled to a 20px font size. The percent unit does change with parent elements.

Control Text: I am unstyled; I have a font-size of 20px.
Unit Text: I am styled to 100%. I am 100% of the 20px font size of this element now.

Web Design Font Units Pros and Cons

The Pixel (px)
Recommend: Print
Avoid: Website (mostly)
Due to its lack of scalability, the popular px may not be the best choice for web design. It can be frusterating for users who may require larger sized fonts but are relucant to zoom in and out to read text.
REM
Recommend: Consistency on Web
Avoid: Using ONLY rem
Rem is scalable by the root font size, but will not scale simply because it is put into a div with a pre-declared font-size. This creates consistency. Because of it's consistency, however, all rem webpages may need more css to adjust padding or margins when the rem size changes (i.e. between H1 and H2, for example).
EM
Recommend: Scalability
Avoid: Using ONLY em
EM is incredibly scalable as it will inherit sizing from parent elements. This creates a modular effect. However, a website created in all EM may require css to adjust padding or margins when elements change sizes.
Percent
Recommend: Reliability
Avoid: Using ONLY percentage
Percent sizing is very similiar to EM, but will almost automatically prevent text from being too small. An EM element that is nested too many times will become unreadable faster than a percent unit. Therefore, percents are more reliable but less scalable.
Viewports
Recommend: N/A
Avoid: Everything
No one is a fan of viewports. Viewports will literally keep the same size text no matter if the user zooms in or out. So they will get the same view at 5% as they would at 300%. For developers, it ensures that the page will look good no matter the zoom of the user, but it can be highly frusterating for users.

Wrap Up

There doesn't seem to be an obvious winner. The best solution seems to be to use REM but add in EM for aspects such as padding, margins, etc. However, with enough CSS the same results can be acheived with any of the units. If you are dead-set on using a unit type by itself that is responsive to parent elements, percent would be better than em. However, percent by itself will not work as well as a REM/EM combo.
